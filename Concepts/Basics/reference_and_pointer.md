# 참조(Reference)와 포인터(Pointer)

지금 단계에서는 포인터보다 참조를 먼저 이해하는 것이 중요하다.
포인터와 메모리 관리는 나중에 따로 정리한다.

---

## 값 복사 vs 참조

변수를 다른 변수에 대입하면 기본적으로 **복사**가 일어난다.

```cpp
int a = 10;
int b = a;  // b는 a의 복사본

b = 20;
cout << a << "\n";  // 출력: 10 (a는 변하지 않음)
```

`&`를 붙이면 복사 대신 **원본을 직접 참조**한다.

```cpp
int a = 10;
int& c = a;  // c는 a의 별명 (같은 값을 가리킴)

c = 20;
cout << a << "\n";  // 출력: 20 (a도 같이 변함)
```

---

## auto& — 컨테이너 순회할 때 사용

`map`, `vector` 같은 컨테이너를 순회할 때 `auto&`를 쓰면
원소를 복사하지 않고 원본에 직접 접근한다.

```cpp
map<int, int> m = {{1, 3}, {2, 1}};

for (auto& p : m) {
    cout << p.first << ": " << p.second << "\n";
    // p.first  = 키 (숫자 값)
    // p.second = 값 (등장 횟수)
}
```

지금은 **"map이나 vector를 순회할 때는 `auto&`를 쓴다"** 정도로만 알아두면 충분하다.

---

## 포인터는 나중에

포인터(`*`)는 메모리 주소를 직접 다루는 개념으로,
언리얼 엔진에서 액터나 컴포넌트를 다룰 때 본격적으로 등장한다.
지금은 `->` 기호를 보면 "포인터로 멤버에 접근하는 것" 정도만 알아두자.

```cpp
// 언리얼에서 자주 보이는 형태 — 지금은 눈에 익히는 것으로 충분
AActor* MyActor = ...;
MyActor->SetActorLocation(FVector(0, 0, 100));
```
